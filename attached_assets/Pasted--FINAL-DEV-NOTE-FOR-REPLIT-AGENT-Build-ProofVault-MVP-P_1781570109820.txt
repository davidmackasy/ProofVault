# FINAL DEV NOTE FOR REPLIT AGENT

## Build ProofVault MVP — Premium iOS Purchase Protection App

I want you to build the first MVP of an app called **ProofVault**.

ProofVault is a **purchase protection app**, not a basic receipt scanner, not an expense tracker, and not an accounting app.

The app helps users save proof of expensive purchases, extract receipt/order details with AI, track return deadlines, track warranty expiry dates, and get reminders before they lose the chance to return, repair, or claim warranty.

The main object in the app is a **Purchase Item**, not a receipt.

A receipt is only one part of the item’s **Proof Pack**.

Everything in the app should be built around this product structure:

**Purchase Item → Proof Pack → Return Deadline → Warranty Deadline → Protection State → Reminder**

If the app starts feeling like a folder of receipts, that is wrong.

---

## 1. Product Positioning

ProofVault should feel like a **premium fintech/wallet app**.

It should not feel like:

* a document scanner
* an accounting app
* a budgeting app
* a tax app
* a business expense tracker
* a generic receipt storage app

It should feel like:

> “Everything I bought is protected, organized, searchable, and ready when I need proof.”

The emotional value is:

> “Can I still return this?”
> “Is this still under warranty?”
> “Where is my receipt?”
> “Can I prove I bought this?”

The app should open on **value and reassurance**, not just a camera.

---

## 2. Core MVP Promise

The MVP should allow a user to:

1. Sign up or sign in.
2. Add a purchase.
3. Scan/upload a receipt, order screenshot, item photo, warranty card, or proof file.
4. Use AI/OCR to extract purchase details.
5. Review and edit the extracted details before saving.
6. Set or confirm return deadline.
7. Set or confirm warranty expiry date.
8. Save a Proof Pack for each item.
9. See what purchases are still returnable.
10. See what warranties are ending soon.
11. Get local notifications before deadlines expire.
12. Search and filter saved purchases in a Vault.
13. View item details and proof files.

## 16. Data Model

Use this basic TypeScript model.

```ts
type PurchaseStatus =
  | "returnable"
  | "ending_soon"
  | "under_warranty"
  | "needs_proof"
  | "expired"
  | "archived"
  | "fully_protected";

type ProofType =
  | "receipt"
  | "online_order"
  | "item_photo"
  | "serial_number"
  | "box_photo"
  | "warranty_card"
  | "notes";

type PurchaseItem = {
  id: string;
  userId: string;

  itemName: string;
  itemDescription?: string;
  storeName?: string;
  category?: string;

  purchaseDate?: string;
  purchaseTime?: string;

  totalAmount?: number;
  currency?: string;

  receiptNumber?: string;
  orderNumber?: string;

  returnDeadline?: string;
  warrantyExpiry?: string;

  status: PurchaseStatus;
  proofComplete: boolean;

  proofPack: ProofFile[];

  notes?: string;

  sourceType: "physical_receipt" | "online_order" | "manual" | "unknown";

  extractionResult?: ExtractionResult;

  createdAt: string;
  updatedAt: string;
};

type ProofFile = {
  id: string;
  itemId: string;
  userId: string;
  type: ProofType;
  fileUrl?: string;
  thumbnailUrl?: string;
  textValue?: string;
  createdAt: string;
};

type ExtractionResult = {
  confidence: number;
  rawExtractedText?: string;

  suggestedStoreName?: string | null;
  suggestedItemName?: string | null;
  suggestedPurchaseDate?: string | null;
  suggestedPurchaseTime?: string | null;
  suggestedTotalAmount?: number | null;
  suggestedCurrency?: string | null;
  suggestedReceiptNumber?: string | null;
  suggestedOrderNumber?: string | null;
  suggestedReturnPolicyText?: string | null;
  suggestedWarrantyText?: string | null;

  needsReview: boolean;
};
```

## 19. AI Extraction Flow

The AI extraction must happen only on the backend.

Flow:

1. User scans/uploads proof.
2. App uploads file to Firebase Storage.
3. App creates extraction request.
4. Firebase Cloud Function reads the file.
5. Cloud Function calls OpenAI vision/OCR.
6. AI returns structured JSON.
7. Cloud Function saves extraction result.
8. App opens AI Extraction Review.
9. User edits/approves data.
10. User saves Purchase Item.
11. App schedules local notifications if deadlines exist.

Never put the OpenAI API key in the app.

---

## 20. AI Extraction Prompt

Use this inside the Cloud Function:

```txt
You are a purchase-proof extraction engine.

You are given an image or PDF that is proof of a purchase. It may be a paper receipt, online order screenshot, email receipt screenshot, warranty card, or confirmation page.

Extract the purchase details and return ONLY valid JSON. Do not include prose, markdown, or explanations.

Return this JSON shape:

{
  "storeName": string | null,
  "purchaseDate": string | null,
  "purchaseTime": string | null,
  "totalAmount": number | null,
  "currency": string | null,
  "items": [
    {
      "name": string,
      "price": number | null
    }
  ],
  "sourceType": "physical_receipt" | "online_order" | "unknown",
  "receiptNumber": string | null,
  "orderNumber": string | null,
  "returnPolicyText": string | null,
  "warrantyText": string | null,
  "confidence": number,
  "rawExtractedText": string
}

Rules:
- If a field is not clearly visible, return null.
- Do not guess aggressively.
- Do not invent a store, date, total, return window, or warranty.
- Null is better than wrong.
- Lower confidence for blurry, cropped, partial, or unclear images.
- Return confidence between 0 and 1.
```

The user must always review extracted details before final save.

## 22. Protection State Logic

Calculate status based on deadlines and proof.

Suggested rules:

* If user archives item → archived
* If no receipt/order proof exists → needs_proof
* If return deadline is active and within 7 days → ending_soon
* If return deadline is active → returnable
* If warranty is active and within 30 days → ending_soon
* If warranty is active → under_warranty
* If proof is complete and deadlines are valid → fully_protected
* If all deadlines are expired → expired

Keep framing positive.

Avoid a global “bad score.”
